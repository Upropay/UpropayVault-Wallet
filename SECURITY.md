# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.0.x   | ✅ Yes              |
| 1.0.x   | ✅ Yes              |
| < 1.0   | ❌ No               |

## Reporting a Vulnerability

We take the security of UV Vault very seriously. If you discover a security vulnerability within this project, please report it privately to us.

**Please do not open a public GitHub Issue for security vulnerabilities.**

### How to report:
1. Send an email to security@upropay.vip.
2. Include a detailed description of the vulnerability.
3. Provide steps to reproduce the issue.
4. If possible, include a proof-of-concept (PoC).

### Our Commitment:
- We will acknowledge receipt of your report within 48 hours.
- We will provide a timeline for the fix.
- We will credit you for the discovery (unless you wish to remain anonymous).

## Security Best Practices
- **Never share your mnemonic phrase or private key.**
- Use a strong, unique password for your local database.
- Keep your operating system and UV Vault software up to date.
- Only download UV Vault from the official website or GitHub repository.
